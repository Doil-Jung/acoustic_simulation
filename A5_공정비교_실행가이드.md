## 2. 학습 명령 (둘 다 동일 레시피)

두 명령의 `dropout / weight_decay / lr / epochs / mode`가 같음 → 차이는 **set vs rnn(순서 정보)** 하나뿐.

### RNN (detach off + 정규화)

```
python v2\train.py --model rnn --no_detach --dropout 0.4 --weight_decay 1e-3 --lr 2e-4 --name rnn_reg --epochs 30 --workers 4
```

### CNN (set, 완전히 같은 레시피)

```
python v2\train.py --model set --dropout 0.4 --weight_decay 1e-3 --lr 2e-4 --name set_reg --epochs 30 --workers 4
```

- 첫 줄에 `device=cuda` 뜨는지 확인.
- 순서는 상관없음. 한 번에 하나씩.

---

## 3. 주의

- **두 컴에서 동시에 같은 `--name`으로 돌리지 말 것** (OneDrive 충돌 사본 발생). 한 컴에서만.
- 레시피 값(dropout 0.4 / wd 1e-3 / lr 2e-4)은 **시작점**. 목표 = epoch-1 정점을 뒤로 밀어 val 바닥을 기존 0.0089 아래로.
  - val 바닥이 안 내려가고 train도 높음 → 과정규화: `--dropout 0.3 --lr 3e-4`로 완화.
  - 여전히 train ≪ val 갭 → 더 세게: `--dropout 0.5`.
