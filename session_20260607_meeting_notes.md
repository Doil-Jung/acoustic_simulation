- 동일한 접근(물 채움 음향 → 전체 r(z) 프로파일 복원)을 한 선행 연구는 검색 범위 내 발견되지 않음. **이 연구가 최초일 가능성 높음.**

## 2. 충북 전람회 이후 진행된 실험 상황

### 실제 녹음 vs 시뮬레이션 비교 실험 수행
- 마이크 종류/위치/각도에 따라 실제 녹음 데이터의 주파수 응답이 크게 달라짐
- **지향성 마이크** 실험 결과:
  - 포먼트 **위치(주파수)**: 시뮬레이션과 **일치** ✅
  - 포먼트 **세기 비율**: **불일치** ❌ (지향성 마이크의 주파수 특성이 원인)
- **결론**: 무지향성 마이크(Earthworks E50)로 전환 필요

### 전달함수 측정 방법
- 스피커 직접음의 영향을 제거하기 위해:
  - **컵 있을 때** 녹음 vs **컵 없을 때** 녹음
  - 둘을 빼서 **컵의 전달함수**만 추출

## 3. 오늘 논의된 핵심 기술적 이슈들

### 3-1. 컵 입구에서 딥(dip)이 나오는 이유

**결론: 물리적으로 정상이다.**

- 폐관-개관 튜브의 공명 모드에서, **개방 끝(입구) = 압력 노드(최소)**
- 모든 공명 모드가 입구에서 압력 노드를 가짐 → 압력 마이크로 측정하면 피크가 아니라 **딥**이 관찰됨
- 마이크를 안쪽이나 바깥으로 옮기면 피크가 보이지만, **모드마다 감도가 들쭉날쭉해지는 문제** 발생

### 3-2. 마이크 위치 전략: 두 가지 선택지

| | A. 옮겨서 피크 보기 | B. 입구에서 딥 보기 |
|---|---|---|
| 신호 강도 | 강함 (피크) | 약함 (딥) |
| 모드 일관성 | ❌ 위치별 불균형 | ✅ 모든 모드 동일 |
| 스윕으로 SNR 보완 | 불필요 | ✅ 가능 |
| Sim-to-Real 난이도 | 어려움 (방사장 모델링 필요) | **쉬움 (입구면 압력만 계산)** |
| 재현성 | 지그로 해결 가능하나 민감 | 입구면 = 명확한 물리적 기준 |

**결정: B를 먼저 시도하기로 함.**
- 이유: Sim-to-Real 정합이 훨씬 간단하고, 스윕 신호로 SNR 보완 가능
- 만약 SNR 부족하면 그때 A로 전환

### 3-3. 전달함수 역수로 딥→피크 변환?

**결론: 안 된다.**
- 딥 지점은 신호가 작음 → 작은 수로 나누면 노이즈가 폭발적으로 증폭
- 수학적 변환으로는 물리적 SNR 한계를 극복할 수 없음

### 3-4. 시뮬레이션 확장 필요성

| 방법 | 마이크 위치 효과 | 교차 모드 | 구현 난이도 |
|---|---|---|---|
| 1D TMM + 방사 모델 | 축 위 거리/각도 | ❌ | 중 |
| 축대칭 2D FEM | 축대칭면 임의 위치 | ✅ | 상 |
| 3D FEM | 임의 3D 위치 | ✅ | 최상 |

- 교차모드 컷오프: f ≈ 1.841c/(πD). 직경 8cm 컵 → ~2,500 Hz 이상에서 1D TMM 부정확
- 현재 학습 범위가 50~5,000 Hz이므로 고주파 대역에서 교차모드 영향 있음
- **결론**: B 방향(입구 딥)으로 가면 복잡한 방사장 모델링이 불필요해지므로, 당장은 TMM 확장 급하지 않음

- 시뮬레이션 출력을 실측과 동일한 형태(딥)로 생성

### Phase 9: PINN (Physics-Informed Neural Network) 도입

## 5. 🚨 긴급 수정 필요 사항: 스윕 신호 처리 방식

### 현재 학생들의 코드 (잘못된 방식)
```
스윕 녹음 → 시간 평균 → FFT → 주파수 응답
= 화이트 노이즈를 넣고 FFT 한 것과 동일
= 스윕의 SNR 이점을 전혀 활용 못함 ❌
```

### 올바른 방식 (디컨볼루션)
```
스윕 녹음 y(t) → FFT(y) / FFT(x) → IFFT → 임펄스 응답 h(t) → FFT → H(f)
                     ↑ x(t) = 알고있는 입력 스윕 신호
→ 매치드 필터 원리로 SNR ~40 dB 향상 (2초 스윕, 5kHz 대역 기준)
→ 추가 보너스: 로그 스윕 사용 시 스피커 비선형 왜곡(고조파)도 시간축에서 분리 제거 가능
```
