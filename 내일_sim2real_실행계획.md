# Sim-to-Real 실행 계획 (내일, 다른 데스크톱)

## 0. 목표와 원칙

- **최종 목표**: 시뮬레이션으로 학습한 모델로 **실제 녹음 데이터**의 형상(반지름 프로파일 r(z) + 높이 H) 추정.
- **원칙**: 시뮬 최고 성능이 목표가 아님 — 시뮬에서 마지막 정밀도를 쥐어짜는 건 **시뮬레이터에 과적합**(reality gap)이라 실측에서 오히려 나빠질 수 있음. 그래서 **실측 평가를 일찍 끌어들여 방향키로 삼는다.**
- **링 장치 = 라벨 있는 실측 앵커** (링 반지름 {10,16,22,28,34,40}mm, 스택 높이 기지). 맨바닥 재학습이 아니라 **sim-to-real 튜닝/검증**용.
- 물리 기반 방법(continuity / PINN / 테스트타임 정제)은 **라벨 없이 측정 스펙트럼만으로** 작동 → 본질적으로 sim-to-real 도구.

## 1. 지금까지 확정된 사실

| 모델 | r_MAE | H_MAE (보정 후) | 비고 |
|---|---|---|---|
| CNN `base_ratio_v2` | 2.47 | 9.13 → **4.38** | 보정 잘 먹음 |
| RNN detach-off (첫 RNN, **기록 유실**) | 좋음 | **CNN보다 나았음**(기억) | 높이 전문가 계열 |
| RNN detach-on `rnn_ratio` | **1.74** | 12.1 → 11.7 | 보정 무력(산포). detach가 높이 굶김 |
| RNN nodetach `rnn_ratio_nodetach` | 2.22 | raw 6.2 (ep1) | [TEST]·보정 미산출(19ep 중단), ep1 저장본 있음 |
| RNN+continuity `rnn_continuity` | **~1.5–1.8** | ~11 | **30ep 안정(과적합 안 터짐), 반지름 최고**. H는 손실균형 탓 |
| dropout/wd/lr 스윕(`*_reg`, `d128`) | — | — | 전부 과적합 폭발. **실패** |

**핵심 교훈**
1. **detach OFF**가 높이에 유리(높이머리가 트렁크 공유 → 상관 좋음 → 보정 먹힘). detach ON은 산포라 보정 무력.
2. **물리 기반 continuity 손실**이 일반 정규화(dropout/wd)보다 과적합·반지름에 훨씬 효과적.
3. **선형 보정은 스케일·바이어스만 펴고 산포는 못 편다** — CNN엔 통했고(9.13→4.38) detach-on RNN엔 안 통함(12.1→11.7).
4. continuity(`v2/models/continuity.py`)는 이미 **PINN 손실 ②(부피 연속방정식)의 경량판**. 즉 물리 정보 손실.

## 2. 선택한 전략: 전문가 결합 + 테스트타임 정제

```
[H 전문가 모델] ─→ H (보정)
                        \
                         ├─ 결합 (r, H) ─→ [테스트타임 정제] ─→ 최종 형상
                        /     (초기값)      미분가능 TMM으로 봉합
[r 전문가=continuity] ─→ r(z)
```

- **전문가 결합**: 시소의 근원(r·H가 트렁크 공유하며 다툼)을 **공유 트렁크 제거**로 근본 회피. detach는 반쪽 격리, 이건 완성판.
- **유일한 약점 = 두 출력의 물리적 불일치** → **테스트타임 정제가 (r,H)를 측정 스펙트럼에 맞춰 최적화하며 자동 봉합.** (정제 안 쓸 때만 continuity 일관성 항 추가.)
- **PINN(순방향 손실 학습 내재)은 나중** — 실측 무라벨 데이터 대량 + 빠른 amortized 추론이 필요해질 때.

## 3. 내일 실행 순서

### Step 0 — 사전 점검 (다른 컴)
- 코드 동기화 확인: `python v2\train.py --help` → `--dropout` 등 보이면 최신.
- CUDA 파이썬: `python -c "import torch;print(torch.__version__,torch.cuda.is_available())"` → `+cu... True`. **`.venv` 쓰지 말 것.**
- `dataset\v2` 우클릭 → "이 장치에 항상 유지".
- (상세는 `A5_공정비교_실행가이드.md` 참조)

### Step 1 — 전문가 두 모델 확보
- **r 전문가**: `rnn_continuity` (이미 있음, 재학습 불필요).
- **H 전문가**: 깨끗한 보정 후 [TEST]를 위해 no_detach로 완주 1회:
  ```
  python v2\train.py --model rnn --no_detach --name rnn_h --epochs 30 --workers 4
  ```
  (또는 기존 `rnn_ratio_nodetach.pt` ep1 저장본 재사용 — 단 [TEST]·보정계수는 없음)

### Step 2 — 결합 + 평가 유틸  *(코드 작성 필요)*
- H(보정) ← H 전문가, r(z) ← continuity 모델 → 결합 (H, r).
- 시뮬 test셋에서 보정 후 H_MAE, 슬롯별 r_MAE 출력.

### Step 3 — 테스트타임 정제 루프  *(코드 작성 필요)*
- 입력: 결합 (r,H) 초기값 + 측정 스펙트럼.
- 손실: ‖TMM(r,H) − 측정 스펙트럼‖ (+옵션 continuity).
- (r,H)에 대해 gradient 몇 스텝. 미분가능 TMM = `v2/forward/tmm.py`.
- 검증: 정제가 시뮬 test 지표를 개선하는지(특히 어려운 샘플·연속 컵).

### Step 4 — 링 실측 검증 (리그 준비되면)
- 기지 링 구성 측정 → 결합+정제 파이프라인 통과.
- **진단**: 정제가 형상을 **기지 링 기하에서 멀어지게** 끌면 = NN 오차가 아니라 **솔버 불일치**(마이크 모델·4.2kHz↑ 반경 모드·리그 효과).
- 실측(기지 높이)으로 **H 보정계수 재적합** (시뮬 보정계수는 실측에 안 맞을 수 있음).

## 4. 미결 / 백로그

- **per-step continuity**: 프로토콜 v2(딥 이동으로 Δh 독립 측정) 도입 시 non-trivial한 강한 제약. v1(고정 ΔV, 수위 미측정)에선 누적형과 중복이라 보류.
- **PINN 순방향 손실(③)**: 실측 무라벨 적응 + 빠른 추론 필요 시. 단점 = 학습 매우 느림(솔버 in-loop×128스텝), 역전파 불안정(공명 피크), 손실가중 민감, 순수 시뮬에선 라벨과 중복.
- **A5 공정 비교**(set vs rnn, 같은 정규화 레시피): 연구/논문용. 배포 파이프라인과 **별개 트랙**.
- **continuity 모델 높이 균형**(c_weight↓/h_weight↑): 결합 방식에선 **불필요** — H는 전문가에서 가져오므로.
