## 0. 이 단계가 증명해야 하는 것

1. **순서 정보의 기여**: 물채움 시퀀스를 시계열로 읽으면(셋 풀링 대비) 형상 오차가 줄어드는가.
2. **시소의 구조적 해소**: CNN 단계에서 규명한 H-r 멀티태스크 간섭(옵티마이저 처방 불응, 눈금 기울기 0.84→~1.12 반전)이 **H 헤드 gradient 분리**로 사라지는가.

두 가지가 분리 검증되도록, 시소 대응과 RNN 도입을 **각각 스위치로** 구현한다(아래 §4 어블레이션).

---

## 1. 구조 (베이스라인과의 차이 최소화)

```
X(B,S,F) ──StepEncoder(공유, baseline과 동일)──► feat(B,S,d)
V(B,S)  ──v_emb──────────────────────────────►(concat)─ step_mix ─► (B,S,d)
                    │
                    ▼
        BiLSTM(d→d/2×2방향, 1~2층, step_mask 기반 pack)
                    │
        ┌───────────┴─────────────┐
   시퀀스 요약 z_seq          per-step 출력(선택, §5)
   (마지막 유효 스텝 h_T      
    + 마스크드 mean/max 병합)  
        │
   trunk(공유 몸통)
        │
   ┌────┴──────────────┐
 head_r(25슬롯)     head_h(높이)
                    ▲ z_seq.detach() 입력   ← 시소 대응 핵심
```

- **StepEncoder·v_emb·step_mix·전처리·정규화·손실·지표 전부 베이스라인 재사용.** 바뀌는 것은 풀링(셋 → BiLSTM)뿐. → 사다리 비교가 "순서 정보" 하나로 귀속됨.
- **BiLSTM 채택 근거**(프레임 §8): 바닥 형상은 초기(긴 공기기둥) 스텝에서만 보임 → 역방향 경로가 바닥 반지름 오차를 줄일 것으로 물리 예측. 단방향/역방향/양방향은 어블레이션 축.
- 가변 길이는 기존 step_mask 그대로: `pack_padded_sequence`로 패딩 스텝이 상태에 안 섞이게.

## 2. 시소 대응 (이번 단계에 내장)

- **주 처방 — H 헤드 gradient 분리**: `head_h(z_seq.detach())`. 몸통·인코더의 gradient는 r 과제가 독점 → 두 과제가 몸통을 서로 당기는 간섭이 원천 차단. H 헤드는 동결된 특징 위의 얕은 회귀자(선형 또는 2층 MLP)로 학습.
  - 근거: CNN 진단에서 H 정보는 특징에 이미 충분(산점도 상관 0.9977). 문제는 눈금 동결 시점 — gradient를 끊으면 H 헤드는 자기 loss만으로 눈금을 끝까지 펼 수 있고, r은 방해받지 않음.
  - 부작용 감시: r 전용이 된 몸통이 H에 불리한 특징을 지울 가능성 → val H 성분 로그로 감시(수치가 CNN 대비 크게 나빠지면 보조 처방으로 전환).
- **보조 처방(어블레이션 축)**: Kendall 불확실성 가중(학습형 log σ² 2개로 h_weight 대체). 주 처방과 상호배타 아님, 비교용.
- **안전망 유지**: `_bestH.pt` 별도 저장, 검증셋 H 선형 보정 자동 내장(train.py 기존 로직), 성분 로그 CSV(`{name}_trainlog.csv`)로 시소 여부 판정.
- **h_weight**: gradient 분리 시 H loss가 몸통에 안 흐르므로 의미 축소 → head_h 학습률만 별도 지정(기본 몸통과 동일, 필요시 상향).

## 3. 학습 설정

- 데이터: 기존 10만 샘플 그대로(재생성 불필요). 스텝 서브샘플링 증강(§9.2) 유지 — RNN이 스텝 번호 지름길을 쓰지 못하게 하는 효과가 시계열 모델에서 더 중요해짐.
- 옵티마이저: AdamW + 코사인(베이스라인과 동일 조건 유지 — 사다리 비교 변인 통제).
- 체크포인트/로그: train.py 공통 사용(모델 클래스만 교체 가능하게 `--model {set,rnn}` 인자 추가).
- 이름 규약: `rnn_ratio` (+ 어블레이션 접미사: `_uni`, `_rev`, `_kendall`, `_nodetach`).

## 4. 어블레이션 계획 (발표 자료 겸용, 프레임 §8 항목 구체화)

| # | 실험 | 비교 대상 | 판정 지표 |
|---|------|-----------|-----------|
| A1 | 셔플 테스트 | rnn_ratio vs 스텝 순서 셔플 입력 | r_MAE 악화량 = 순서 정보의 기여 |
| A2 | 방향 | 단방향/역방향/양방향 | 구간별(상/중/하부 슬롯) r 오차 — 역·양방향이 **바닥** 오차를 줄이는지(물리 예측 검증) |
| A3 | 요약 방식 | 마지막 스텝 only vs mean/max 병합 vs 어텐션 | r_MAE, H_MAE |
| A4 | 시소 처방 | detach(주) vs Kendall(보조) vs 무처방 | val 성분 로그의 시소 진폭, 보정 전 H_MAE(눈금이 스스로 펴졌는가), 보정계수 a가 1에 접근하는가 |
| A5 | 사다리 본 비교 | CNN 확정본 vs RNN 확정본 | 동일 테스트셋 r_MAE/H_MAE (+구간별) |

- **성공 기준(합격선)**: A5에서 r_MAE가 CNN 2.47mm 대비 유의하게 감소(특히 하부 슬롯), A4에서 시소 진폭 감소 + 보정계수 a→1 접근.
- A4가 이 연구 서사의 핵심 증거: "진단(직선·시소) → 처방(구조 분리) → 완치(눈금 자가 회복)"의 완결.

## 5. 이후 확장 여지 (이번 단계에서 하지 않는 것)

- per-step 출력(스텝별 수위 h(t) 예측)은 PINN 단계의 연속방정식 잔차 손실에서 도입 — RNN 단계에서는 §7 규격(H + 25슬롯)만 출력해 비교 순수성 유지.
- 어텐션 요약이 A3에서 이기더라도 Transformer 전환은 하지 않음(사다리 서사 유지, 스텝 수 ≤128이라 이득 제한적).
